Use Swift Package Manager with iOS projects

Dec 04 2018

The Swift package manager, or SwiftPM, currently has no support for iOS platforms, however, it is still possible to use it for this purpose.

How?

  1. Initialise a library package: 
    $ swift package init --type library
  2. Create a xcode.xcconfig file, that will specify the platforms you want to support. For example: 
    SUPPORTED_PLATFORMS = iphoneos iphonesimulator
IPHONEOS_DEPLOYMENT_TARGET = 9.0
  3. Then generate .xcodeproj file using this config: 
    $ swift package generate-xcodeproj --xcconfig-overrides xcode.xcconfig
  4. Create a workspace and add the generated project to it.
  5. Next, create a new project targeting iOS application and add it to the workspace too.
  6. Finally, add the frameworks of the library plus its dependencies as embedded binaries of the iOS project.

That's it!

Any precautions to take?

  • When you create or add a new file you need to take care of the target where the file is added (this could be modify in the Target Membership section of the File Inspector).
  • The code inside the library need to be marked as public to be visible outside of the framework.
